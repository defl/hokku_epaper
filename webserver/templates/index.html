Dithering…

Serpentine error diffusion

What it does: Normally each row is scanned left-to-right, so quantization error is always pushed ahead in the same direction. With serpentine (boustrophedon) scanning, every other row is processed right-to-left, and the error-diffusion pattern is mirrored horizontally so error still flows only into pixels that have not been finalized yet.

Why use it: A single scan direction can leave visible directional streaks or "combing" in smooth areas. Alternating direction breaks that bias and often looks more even, similar to the default recommendation in mature dithering libraries.

Applies to Floyd–Steinberg, Atkinson, and Stucki. Turning it on or off re-converts images (cached results for both settings are kept). Default is off for backward compatibility with earlier renders.
